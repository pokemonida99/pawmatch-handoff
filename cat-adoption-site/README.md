# 合拍 PawMatch｜貓咪認養平台(單頁式 MVP)

依據 [`PROMPTS.md`](./PROMPTS.md) 的「認養網站生成 Prompt 組」實作:**Prompt A 主站 + B1–B5 模組深化 + Prompt C 易用性驗收**,全部整合在單一可運行檔案 [`index.html`](./index.html)。

## 如何執行

兩個檔案內容完全相同,擇一開啟:

- **`index.html`**(可讀原始碼版):React 18 UMD + Babel Standalone + Tailwind CDN,需網路載入函式庫。要改程式改這一份。
- **`index.offline.html`**(離線建置版):JSX 已用 esbuild 預編譯、Tailwind 已預產靜態 CSS、React 全部內嵌——**零網路、零相依,雙擊即開**。已用 Chromium 實測通過完整配對流程。

```bash
# 或起一個本機伺服器
python3 -m http.server 8000 --directory cat-adoption-site
# 瀏覽 http://localhost:8000
```

## 設計計畫(Prompt A 要求的說明)

1. **色票**取自貓毛色系:奶油 `#FBF7F0`(底)、奶茶 `#F0E7D8`(卡片)、暖橘 `#C05621`(全站唯一主 CTA 色,白字對比 4.52:1)、灰藍 `#47586A`(次要)、深炭 `#2F2C28`(文字),輔以苔綠(完成態)與蜜糖黃(提醒態)。
2. **字體刻意配對**:標題用 Zen Maru Gothic(圓體=寵物友善咖啡店的溫度,同時避開「米白底＋襯線大標」的 AI 樣板),內文用 Noto Sans TC 保住長文可讀性。
3. **唯一招牌記憶點**:貓掌腳印進度指示——配對測驗每答一題就多一個實心橘掌印,搭配「還差 N 題就能看結果」的正向句式;全站不再使用第二個裝飾記憶點。
4. **主要取捨**:凡「轉換率」與「被審查感」衝突,一律先消除被審查感——所以沒有「無符合結果」、沒有必填欄位、測驗每題可跳過、「不合適」由系統轉介而非通知原因。
5. **照片以 SVG 毛色漸層佔位**(溫暖日常、無悲情救援影像),單檔零外部圖片相依,換成真實貓照時直接替換 `CatAvatar`。

## Prompt C 驗收紀錄(修正清單:問題 → 修法)

- **禁用詞掃描**:布丁需求卡出現「追蹤腎指數」→ 改為「回診看看腎指數的近況」。全站再掃「審核/資格/申請條件/追蹤/檢查/監督/合格」= 0 筆。
- **觸控目標**:隱私切換膠囊(32px)、頁尾連結與設定展開(36px)未達 44px → 全部提升至 `min-h-[44px]`;主 CTA 48px、測驗選項 52px。
- **轉換與框架**:首屏 5 秒內主張「找到跟你的生活合拍的貓」;主 CTA「開始配對」全站唯一暖橘;副 CTA「先逛逛貓咪」灰藍描邊。
- **流失點**:測驗一屏一題、每題「跳過這題」、進度為貓掌+「還差 N 題」正向句式;全站表單首屏最多 1 個欄位(訊息草稿 textarea)。
- **信任四要素**:中途實名介紹(檔案頁+後台)、健康護照(時間軸)、透明帳本(獨立頁+支援區入口)、無責退回(支援區+回家之後設定)——各出現 ≥1 次;最終判斷權在中途,以「他們的把關是給你和貓的保障」正面解釋。
- **可及性**:語意化 HTML(nav/main/section/ol/table caption)、skip-link、`:focus-visible` 橘色外框、SVG 佔位圖皆有 `role="img"`+`aria-label`、對比 ≥4.5:1(主文字 #2F2C28/#47586A on 奶油底)、`prefers-reduced-motion` 全域停用動畫、測驗進度 `aria-live="polite"`、Modal 支援 Esc 關閉與 aria-modal。
- **行動裝置**:375px 無橫向捲動(帳本表格包在 `overflow-x-auto` 容器內)、行動選單、主 CTA 於首屏拇指可達。
- **錯誤與空狀態**:帳本篩選無結果時顯示「換個月份或類別看看」的行動邀請,不道歉不賣萌。
- **語法驗證**:JSX 以 esbuild 編譯通過,無錯誤。
- **實機驗證**:Chromium 375×812 實測通過——渲染正常、無橫向捲動、配對全流程(含租屋者條件式房東題)、結果頁三筆合拍理由與房東小提醒、「跟中途聊聊」草稿確認後送出,全部符合預期。

## 易用性報告改進落地(2026-07-09)

依 `USABILITY-REPORT.md` Part 6 的優先改進清單,五項全部實作並以 Chromium 實測通過:

- **P0 草稿頁除罪化**:`ChatModal` 草稿上方加「中途看的是你願不願意學,不是你懂多少。」
- **P0 送出隱私說明**:`ChatModal` 送出鈕上方加「這段摘要只寄給(中途名),不會公開、不進相簿。」
- **P1 前移「試相處」**:`Quiz` 結果頁每張結果卡 CTA 下方加「先見面、可試相處,不用一次決定。」
- **P2 無責退回入口前移**:`Journey` 旅程末「中途把關」說明卡加無責退回錨點連結(#support)。
- **P2 相簿導流**:`Album` 每張近況卡加「像牠一樣的貓,正在找家 →」連結(#cats);中途後台標記「不合適」後顯示系統轉介回饋,認養人端送出成功畫面同步告知「不會空手而回」。

**附帶修復**:`index.html` 的 Babel Standalone CDN 未鎖版本,unpkg 升到 Babel 8 後(preset-react 預設 automatic runtime、輸出 `import` 語句)造成整站白屏——已鎖回 `@babel/standalone@7`。`index.offline.html` 已依新內容重建。禁用詞重掃 = 0 筆。

## 研究發現 → 功能對照(落地版)

| 研究發現 | 本站落地位置 |
|---|---|
| 長問卷造成前端流失 | `Quiz`:一屏一題、可跳過、貓掌正向進度 |
| 共感型認養人退養最低 | `CatCard`/`CatProfile`:第一人稱個性敘事置頂 |
| 守門製造對立 | `matchCats`:無不及格、永遠輸出「目前最合拍」 |
| 中途需保有最終判斷權 | `Journey` 正面說明+`AdminPage` 三態標記與系統轉介 |
| 回報的監視感 | `AdopterView`:曬貓語言、抽獎池、隱私預設只給中途 |
| 退養隱瞞比退養更糟 | 無責退回入口(支援區+設定),原因非必填 |
| 信任稀缺 | `LedgerPage` 公開帳本、健康護照、中途實名 |
